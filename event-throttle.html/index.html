EventThrottle Demo

Character Count:0 Word Count:0

This demonstration uses two EventThrottle instances:

The first specifies a throttle duration of 500 milliseconds and raises an event used to calculate and update the character and word counts of the text-area as the user types.

The second specifies a throttle duration of 1,500 milliseconds and the suppress active1 option, and raises an event used to restyle the text-area after the user has stopped typing.

1 Setting the suppressActive property of the EventThrottleOptions object used to configure the EventThrottle instance to true results in only the last of a sequence of upstream (source) events being forwarded. In practise, this means that the EventThrottle instance will only dispatch a throttled event if no additional upstream events were raised after the active event was queued for processing.
